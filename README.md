# CatCafe
#ที่มาของปัญหาและความสำคัญ
ปัจจุบันคาเฟ่ที่ให้บริการทั้งอาหาร เครื่องดื่ม แมว และบอร์ดเกม มักมีการจัดการข้อมูลแบบแยกส่วนหรือบันทึกด้วยวิธีแบบเดิม เช่น จดบันทึกด้วยกระดาษหรือใช้ไฟล์ทั่วไป ส่งผลให้เกิดความผิดพลาดในการคิดค่าบริการ การติดตามการเข้า–ออกของลูกค้า การรับออเดอร์ และการยืมคืนบอร์ดเกม
ดังนั้นการพัฒนาระบบบริหารจัดการคาเฟ่แมวและบอร์ดเกมจึงมีความสำคัญ เพื่อช่วยให้การจัดการข้อมูลลูกค้า พนักงาน เมนู การใช้บริการ และการชำระเงินเป็นระบบ มีความถูกต้อง และลดภาระงานของพนักงาน
#จุดประสงค์ของโครงการ
เพื่อพัฒนาระบบสำหรับจัดการข้อมูลลูกค้า พนักงาน และแมวภายในร้าน
เพื่อจัดการข้อมูลการเข้าใช้บริการ (Check-in / Check-out) ของลูกค้า
เพื่อรองรับการสั่งอาหาร เครื่องดื่ม และการยืมบอร์ดเกม
เพื่อช่วยคำนวณค่าใช้บริการและบันทึกการชำระเงินอย่างถูกต้อง
#ประโยชน์ที่คาดว่าจะได้รับ
ช่วยให้การจัดการข้อมูลภายในร้านมีความเป็นระเบียบ
ลดความผิดพลาดในการคำนวณค่าใช้บริการและค่าอาหาร
เพิ่มความสะดวกและรวดเร็วในการทำงานของพนักงาน
สามารถตรวจสอบข้อมูลย้อนหลังได้ เช่น ประวัติการเข้าใช้บริการและการชำระเงิน
ขอบเขตของโครงการ (Scope of Project)
#สิ่งที่ระบบทำได้
จัดการข้อมูลลูกค้า พนักงาน แมว เมนู และบอร์ดเกม
บันทึกการเข้า–ออกใช้บริการของลูกค้า
บันทึกการสั่งอาหารและรายละเอียดการสั่งซื้อ
บันทึกการยืม–คืนบอร์ดเกม
คำนวณและบันทึกการชำระเงิน
#สิ่งที่ระบบไม่ทำ
ไม่รองรับการจองล่วงหน้าผ่านระบบออนไลน์
ไม่รองรับการชำระเงินผ่านระบบธนาคารหรือแอปภายนอก
ไม่รองรับระบบสมาชิกแบบสะสมแต้ม
#Functional Requirements
ระบบสามารถเพิ่ม แก้ไข และลบข้อมูลลูกค้าได้
ระบบสามารถบันทึกเวลาเข้าและออกของลูกค้าได้
ระบบสามารถบันทึกการสั่งอาหารและจำนวนที่สั่งได้
ระบบสามารถบันทึกการยืมและคืนบอร์ดเกมได้
ระบบสามารถคำนวณค่าใช้บริการและบันทึกการชำระเงินได้
#Non-Functional Requirements
ระบบต้องใช้งานง่ายและเข้าใจได้ง่าย
ระบบต้องตอบสนองได้รวดเร็ว
ระบบต้องมีความถูกต้องของข้อมูล
ระบบสามารถใช้งานผ่านเว็บเบราว์เซอร์ได้
ระบบมีการจำกัดสิทธิ์การเข้าถึงข้อมูลตามผู้ใช้งาน
