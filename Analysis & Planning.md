# CatCafe
# 1. ศึกษาปัญหาและความต้องการของระบบ
### พบว่าการจัดการข้อมูลภายในร้านคาเฟ่แมวและบอร์ดเกมยังขาดระบบที่เป็นศูนย์กลาง ทำให้เกิดความซ้ำซ้อนของข้อมูล และความผิดพลาดในการบันทึกการเข้าใช้บริการ การรับออเดอร์ และการคิดค่าบริการระบบที่พัฒนาขึ้นจึงต้องสามารถจัดการข้อมูลลูกค้า การเข้า–ออกใช้บริการ การสั่งอาหาร การยืมบอร์ดเกม และการชำระเงินได้อย่างเป็นระบบและถูกต้อง
# 2. กำหนดขอบเขตของโครงการ
### ระบบครอบคลุมการจัดการข้อมูลลูกค้า พนักงาน แมว เมนู บอร์ดเกม การเข้าใช้บริการ การสั่งอาหาร การยืม–คืนบอร์ดเกม และการชำระเงินระบบไม่ครอบคลุมการจองล่วงหน้า ระบบสมาชิกสะสมแต้ม และการชำระเงินผ่านระบบออนไลน์ภายนอก
# 3. ออกแบบโครงสร้างฐานข้อมูล (ER Diagram / .mb)
### ออกแบบฐานข้อมูลโดยแบ่งข้อมูลออกเป็นตารางหลัก ได้แก่ Customers, Staff, Cats, Menu, Visits, Orders, Order_Details, Payments, BoardGames และ Game_Borrowตารางต่าง ๆ เชื่อมโยงกันด้วยความสัมพันธ์แบบ Primary Key และ Foreign Key เพื่อให้สามารถจัดเก็บและเรียกใช้งานข้อมูลได้อย่างถูกต้องและเป็นระบบ
# 4. ระบุ Functional และ Non-Functional Requirements
## Functional Requirements
### ระบบสามารถจัดการข้อมูลลูกค้า พนักงาน และเมนูได้
### ระบบสามารถบันทึกการเข้า–ออกใช้บริการของลูกค้าได้
### ระบบสามารถบันทึกการสั่งอาหารและการยืมบอร์ดเกมได้
### ระบบสามารถคำนวณและบันทึกการชำระเงินได้
## Non-Functional Requirements
### ระบบต้องใช้งานง่ายและเข้าใจได้ง่าย
### ระบบต้องมีความถูกต้องของข้อมูล
### ระบบต้องตอบสนองได้รวดเร็ว
### ระบบสามารถใช้งานผ่านเว็บเบราว์เซอร์ได้
